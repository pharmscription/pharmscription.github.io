Idee hinter dem Projekt

Die Software soll das Ausstellen und Einlösen von Rezepten vereinfachen. Sie ist primär zur Verwendung innerhalb eines Krankenhauses, oder des Krankenhaus Verbandes gedacht. Die Software ermöglicht die Verwaltung von Patienten, Ärzten und Rezepte für Patienten. Es soll möglich sein, mit eventuell vorhandenen Benutzer- und Personaldaten zu arbeiten. Ärzte können Rezepte für Patienten, welche in deren Behandlung sind, ausstellen. (Welche, wann, wie viel) Sobald ein Rezept erstellt wurde, sollen alle Änderungen protokolliert werden, es soll die komplette History ersichtlich sein. Der Zugriff, auf Patienten, Rezepte und Ärzte ist eingeschränkt. Apotheker soll es möglich sein herausgegebene Arzneimittel für eine spätere Abrechnung zu vermerken.

Das Ziel während des Engineering Projekts sollte sein, ein Prototyp der Anwendung fertiggestellt zu haben. Es soll das HTTP-Backend und das Web Front-End in einem stabilen Zustand sein. Im Prototyp geht es nur um die Funktionalität, der Datenschutz und Datensicherheit werden nicht berücksichtigt.

Die Applikation soll:

  • Ärzte können Rezepte erfassen
  • Apotheker können Rezepte ansehen und herausgegebene Medikamente erfassen
  • Patienten können ihre ausgestellten Rezepte ansehen
  • Analyse der Medikamenten Verwendung
  • API für Export, zur späteren Anbindung von Versicherungen
